# PR 1 — Foundation: Python ≥3.10 + UV + dependency-groups

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump Speasy's declared Python floor to 3.10, switch dev workflow and CI to UV with `[dependency-groups]` as the single source of truth, eliminate `requirements*.txt` and `tox.ini` duplication, and update Read the Docs and developer docs to match.

**Architecture:** Replace the dual `requirements.txt` + `requirements_dev.txt` + `docs/requirements.txt` setup with `[dependency-groups].dev` and `[dependency-groups].docs` declared in `pyproject.toml`. Add `uv.lock` for reproducible installs. Switch CI (`.github/workflows/tests.yml`) and RTD (`.readthedocs.yaml`) to use `astral-sh/setup-uv` + `uv sync`. Update `Makefile`, `CLAUDE.md`, `CONTRIBUTING.rst`, `README.md` to drop the `PYTHONPATH=.` pattern in favor of `uv sync` + `uv run`. Build backend (`flit_core`) is unchanged in this PR — backend swap is PR 10.

**Tech Stack:** UV ≥0.5, hatchling stays untouched (still flit_core for now), GitHub Actions, Read the Docs.

```toml
[dependency-groups]
dev = [

docs = [
```

Notes:

- `pandoc` is the Python wrapper used by `nbsphinx`; the CI job currently `pip install pandoc`s it inline, so it belongs with the other docs deps.

```yaml
      if: matrix.python-version == '3.10' && matrix.os == 'ubuntu-latest'

        pytest --cov=./ -cov-config=.coveragerc --cov-report=xml

```

with:

```yaml

      uses: astral-sh/setup-uv@v6
      with:
        enable-cache: true
    - name: Install dependencies
      run: uv sync --group dev --group docs
    - name: Lint with flake8
      run: |
        uv run --with flake8 flake8 . --count --select=E9,F63,F7,F82 --show-source --statistics
        uv run --with flake8 flake8 . --count --exit-zero --max-complexity=10 --max-line-length=127 --statistics
    - name: Test with pytest
      if: matrix.python-version != '3.10' ||  matrix.os != 'ubuntu-latest'
      env:
          SPEASY_CORE_HTTP_REWRITE_RULES: '{"https://thisserver_does_not_exists.lpp.polytechnique.fr/pub/":"http://sciqlop.lpp.polytechnique.fr/cdaweb-data/pub/"}'
      run: uv run pytest
    - name: Test with pytest (coverage + long tests)
      if: matrix.python-version == '3.10' && matrix.os == 'ubuntu-latest'
      env:
        SPEASY_AMDA_MAX_CHUNK_SIZE_DAYS: "25"

        SPEASY_INVENTORY_TESTS: ""
        SPEASY_CORE_HTTP_REWRITE_RULES: '{"https://thisserver_does_not_exists.lpp.polytechnique.fr/pub/":"http://sciqlop.lpp.polytechnique.fr/cdaweb-data/pub/"}'

        make doctest
```

Notes:
- `flake8` stays in this PR — it's removed in PR 11 (ruff config). We invoke it via `uv run --with flake8` so it doesn't pollute the dev group.
- `astral-sh/setup-uv@v6` is the canonical action; `enable-cache: true` reuses uv's cache across runs for speed.
- `uv sync --group dev --group docs` installs the project editable + both dependency groups in one call.
